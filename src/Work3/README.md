# CG-Lab3 - Phong & Blinn-Phong Ray Tracing

## 1. 项目简介

本实验实现了一个支持 GPU 加速的实时光线追踪渲染器。脱离了传统 CPU 循环与外部模型，在 Taichi Kernel 中利用数学隐式方程直接定义三维几何体，并实现了完整的局部光照模型。

在完成基础的 Ambient、Diffuse 和 Specular 计算之上，搭建了现代化的 UI 交互面板，并完成了**硬阴影 (Hard Shadow)** 与 **Blinn-Phong 高光模型升级** 两项选做内容。

核心亮点包括：
- **纯 Taichi GPU 渲染**：利用 `@ti.kernel` 和 `@ti.func` 实现了像素级高度并行，达到丝滑的 60+ FPS 实时渲染帧率。
- **隐式几何求交**：实现了射线与球体、有限高度圆锥（含侧面与底面）、无限平面的精确数学求交。
- **深度测试**：正确处理多物体遮挡关系，确保场景渲染的物理正确性。
- **实时 UI 交互**：接入 `taichi.ui.Window`，支持通过滑动条实时无缝调节材质参数。

## 4. 实现功能

### 基础功能
- **隐式场景构建**：
  - 左侧蓝色球体：圆心 `(-1.2, -0.2, 0.0)`，半径 `1.0`。
  - 右侧橙色圆锥：顶点 `(1.2, 1.2, 0.0)`，底面高度 `y = -1.4`，底面半径 `1.0`。
  - 底部灰色地面：高度 `y = -1.4`（用于承载阴影）。
- **光照着色器 (Shader)**：
  - **环境光**：模拟全局多次反射的背景光 `I_ambient = K_a * I_light * C_obj`
  - **漫反射**：基于 Lambert 定律，强度与入射角余弦成正比 `I_diffuse = K_d * max(0, N·L) * I_light * C_obj`
  - **镜面高光**：模拟光滑表面的强光反射，避免了背面高光的非法运算。
- **UI 实时控制面板**：在画面左上角提供交互窗口，可实时拖拽修改 $K_a$, $K_d$, $K_s$ 以及 $Shininess (N)$。
- **效果展示**：
  - **环境光**：模拟全局多次反射的背景光，影响场景的全局亮度。
  - **漫反射**：根据入射角余弦成正比，影响物体受光面的色彩强度。
  - **镜面高光**：模拟光滑表面的强光反射，避免了背面高光的非法运算。

### 选做功能 (Bonus)
1. **硬阴影 (Hard Shadow)**：
   - **实现机制**：在交点处，沿着法线方向微小偏移（防止表面自相交噪点），向光源方向发射一条“阴影射线”。若在到达光源之前击中其他物体（如球体遮挡了圆锥，或物体遮挡了地面），则判定该点处于阴影中，此时该像素只计算环境光分量。
2. **Blinn-Phong 模型升级与对比**：
   - **实现机制**：计算视线 $V$ 与光线 $L$ 的半程向量 $H = normalize(L + V)$，将高光计算从 $max(0, R·V)^n$ 升级为 $max(0, N·H)^n$。
   - **视觉体现**：在 UI 面板中提供了一个 **Toggle Model 按钮**，可一键在 Phong 和 Blinn-Phong 之间无缝切换，直观对比两者在高光区域边缘和形态上的视觉差异。

## 5. 实验分析：Blinn-Phong 与 Phong 的视觉差异

在实验过程中，通过 UI 按钮切换两种模型，可以观察到以下显著差异：

1. **高光形状与柔和度**：
   - **Phong 模型**：高光基于反射向量 $\mathbf{R}$ 与视线 $\mathbf{V}$ 的夹角。其高光点通常较小且边缘锐利，在低 $n$ 值下容易出现不自然的过渡。
   - **Blinn-Phong 模型**：基于法线 $\mathbf{N}$ 与半程向量 $\mathbf{H}$ 的夹角。由于 $\mathbf{H}$ 的变化比 $\mathbf{R}$ 更平缓，其产生的高光区域通常比 Phong 模型更圆润、更广，过渡也更自然。

2. **大入射角（边缘区域）表现**：
   - 在**大入射角（即光线几乎平行于物体表面的边缘位置）**时，Phong 模型的高光衰减非常剧烈，有时会产生明显的截断现象，显得不够真实。
   - Blinn-Phong 模型在大入射角下表现更优，高光能够更平滑地延伸到物体的边缘（掠射角处），这更符合真实物理中微平面反射的特征。
   - **视觉结论**：在大角度观察物体侧面时，Blinn-Phong 能维持一个连贯的高光形状，而 Phong 容易出现高光斑点变形或突然消失。

## 6. 运行方式

### 运行环境与启动
确保环境中已安装 `taichi`。在项目根目录（`CG-lab`）下执行：

```bash
uv run -m src.Work3.main
```

### 交互操作说明

程序运行后，所有操作均通过窗口左上角的 **Control Panel (UI 面板)** 通过鼠标完成：

| UI 控件 | 功能描述 |
|---------|----------|
| **Ambient (Ka) 滑动条** | 调节环境光系数 (0.0 ~ 1.0)，控制暗部的基础亮度。 |
| **Diffuse (Kd) 滑动条** | 调节漫反射系数 (0.0 ~ 1.0)，控制物体受光面的色彩强度。 |
| **Specular (Ks) 滑动条** | 调节镜面高光系数 (0.0 ~ 1.0)，控制反光斑的亮度。 |
| **Shininess (N) 滑动条** | 调节高光指数 (1.0 ~ 128.0)，数值越大，高光光斑越小且越锐利。 |
| **Toggle Model 按钮** | 一键切换当前光照计算模型（在 `Phong` 和 `Blinn-Phong` 之间来回切换）。 |

### 观察与测试建议
1. **测试阴影**：观察地面上以及右侧圆锥上由左侧蓝色球体投射出的黑色物理阴影。
2. **测试高光模型切换**：将 `Shininess` 调低（如 16 左右），点击 `Toggle Model` 按钮，可以明显观察到 Blinn-Phong 模型的高光过渡比传统 Phong 模型更圆润、更符合真实物理表现。
3. **防止过曝**：如果在极端参数下画面发白，是由于光照强度累加超过了 1.0，底层代码已使用 `ti.math.clamp` 进行了颜色安全截断保护。
